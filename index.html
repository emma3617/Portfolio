北京歷史建築查詢系統

關鍵字：WebGIS、Esri APIs and SDKs

2024年2月前往中國北京大學交換，預計6月結束，此作品為「GIS設計與應用」的課堂作業。
專案題目為：北京歷史建築物查詢系統，用戶可透過瀏覽此網頁達到查詢北京市著名古蹟建築的目的，除此之外，也設計進階查詢條件，可利用篩選功能(如：歷史人物關聯、區域、年代、風格等)更便捷使用者。
目前專案仍然再開發測試中，因此功能尚未健全，目前進度到達：

  1. 切換底圖
  2. 基本地點查詢
  3. 地圖漫遊瀏覽
  4. 屬性資料查詢

林口影像分類

關鍵字：影像分類、遙測衛星影像、變遷分析

此作品是學習遙測知識所做的課堂練習，作業目的為使用ArcGIS內建的影像分類功能，分析土地利用變遷差異，影像來源採用Landsat系列衛星影像。
選址範圍為新北市林口區，林口區作為建商與產業園區開發的熱門地點，預期可以在衛星影像上看到明顯的土地利用變遷情況，本次影像選用2000與2020年作為對比，左方顯示的影像已經過分類處理，利用中間swipe功能可以拖曳拉動觀察， 變遷矩陣(單位：pixel)如下：

Text Segment Anything Model 內容說明

關鍵字：Deep Learning、特徵擷取、圖像分割(Image Segmentation)

Esri Analytics在2024年2月3日發佈Text Segment Anything Model(以下簡稱TextSAM)範例模型，可使用文字來提取圖片特徵。
此模型基於IDEA-Research的開源Grounding DINO 和 Meta的Segment Anything Model，這幾日也測試用TextSAM對高解析度的航照進行車輛(Car、Vehicle)、建物(Building)等物體提取，可以看到左邊的初步成效確實蠻不錯的，但confidence仍然過低，因此下一步是研究提高模型信心程度。

Books Recommend Linebot

關鍵字：Linebot、Flask、Docker、MongoDB、OpenAI API

此小型專案主要開發一個主題Linebot，主題內容為書籍推薦，使用者可以透過添加書籍、搜尋偏好關鍵字進行書籍內容的追蹤。
專案使用到的技術有：前端使用Linebot輸入訊息，並套用Line Messaging API用於與LINE平台進行交互；後端資料庫使用MongoDB；中間Flask處理前端發送的請求並與後端交互，並且使用Docker控制開發環境，方便未來更深入的專案開發。主要的交互過程如下：

  1. 使用者發送訊息到 LINE Bot
  2. LINE Bot 接收訊息並發送到 Flask
  3. Flask 處理訊息，根據訊息的內容，Flask 應用決定如何處理，例如：更新使用者偏好、查詢書籍數據、調用 OpenAI API 等。
  4. 如果訊息涉及到數據查詢或更新，Flask會使用 pymongo 與 MongoDB 進行交互；如果訊息是非關鍵詞訊息，Flask 應用會通過 httpx 調用 OpenAI API，生成回應。
  5. 返回結果給使用者，Flask 應用將生成的回應通過 LINE SDK 發送回 LINE 平台，最終顯示給使用者。
